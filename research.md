# Research Notes - InnovatEPAM Portal

## Stack Decisions
1. Choose a stack you already know well.
2. Keep the implementation simple and demo-friendly.
3. Favor maintainable patterns over experimental architecture.

## UX Notes
1. The submission flow should be short and clear.
2. Validation messages should be actionable.
3. Admin review should be easy to scan.

## Security Notes
1. Role checks must happen server-side.
2. File uploads must be validated.
3. Sensitive data should not be exposed in read models.

## Scope Notes
1. MVP should focus on core end-to-end value.
2. Drafts, blind review, and multi-stage workflows can be future phases.
